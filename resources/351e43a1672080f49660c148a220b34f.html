💡 시작하기 전에: 커넥터(Connector)란?

커넥터는 클로드가 외부 서비스나 앱과 연결해서 데이터를 가져오거나 작업을 수행할 수 있게 해주는 MCP(Model Context Protocol) 서버입니다.

쉽게 말해, 클로드가 혼자 일하는 게 아니라 Notion, Slack, GitHub, Google Drive 같은 툴들과 직접 연결되어 함께 일할 수 있게 해주는 다리 역할을 합니다.

예를 들어:

  • GitHub에서 최신 이슈를 가져와 Notion에 자동으로 정리
  • 매일 아침 특정 정보를 수집해서 Slack에 요약 발송
  • 반복적인 리포트 작성, 카드뉴스 제작을 자동화

이 모든 것이 커넥터를 통해 가능합니다!

🖥️ 클로드 데스크톱 앱 + Cowork '예정됨' 기능

클로드 데스크톱 앱을 설치하면 Cowork 기능을 사용할 수 있습니다. 그 중에서도 '예정됨(Scheduled)' 기능은 업무 자동화의 핵심입니다.

예정됨(Scheduled)이란?

원하는 프롬프트를 저장해두고, 매일·매주·매시간 등 원하는 주기로 Claude가 자동으로 실행해주는 기능입니다.

즉, 여러분이 직접 클로드에게 매번 요청하지 않아도, 설정해둔 작업을 클로드가 자동으로 수행하고 결과를 지정된 툴(Notion, Slack 등)에 저장해줍니다.

흐름 요약:

커넥터 설정 → 프롬프트 작성 → 예정됨에 등록 → 자동 반복 실행 ✅

✅ STEP 1. 준비 단계 — 자동화 시작 전, 먼저 이것부터

요약 : 반복 업무를 파악하고 도구를 미리 정해두세요.

01. 자동화할 일 정하기

자동화를 시작하기 전, 먼저 어떤 업무를 자동화할지 명확히 정해야 합니다.

다음 질문을 스스로에게 던져보세요:

  • 매일 또는 매주 반복적으로 하는 작업이 있나요?
  • 데이터를 수집하고 정리하는 루틴 작업이 있나요?
  • 여러 툴 사이에서 정보를 복사·붙여넣기 하는 작업이 있나요?

그 중에서 가장 자동화하고 싶은 업무 하나를 먼저 선택하세요. 처음부터 복잡하게 시작하지 않는 것이 좋습니다!

02. 커넥터(MCP 서버) 고르기

자동화할 업무가 정해졌다면, 필요한 커넥터를 역할별로 나눠서 정해두세요.

커넥터는 크게 두 가지 역할로 나뉩니다:

📥 INPUT — 정보 가져올 도구 (1~2개 권장)

Claude가 작업에 필요한 데이터를 가져오는 역할을 합니다.

예시:

  • Claude in Chrome — 웹 페이지 정보 수집
  • GitHub — 이슈, PR, 코드 정보 가져오기
  • Google Drive — 문서, 스프레드시트 읽기

💡 INPUT 커넥터는 너무 많으면 관리가 복잡해지므로 1~2개를 권장합니다.

📤 OUTPUT — 결과 받아볼 곳 (1개 권장)

Claude가 작업 결과를 저장하거나 전송하는 역할을 합니다.

예시:

  • Notion — 정리된 내용을 페이지로 저장
  • Slack — 요약본을 채널에 자동 발송

💡 OUTPUT 커넥터는 1개로 집중하는 것이 가장 깔끔합니다.

2단계. AI에게 추천 요청하기

커넥터 목록을 제공한 뒤 다음과 같이 물어보세요:

💬 프롬프트 예시
💡
"클로드 커넥터로 [이런 목적]의 자동화를 할건데 이중에서 어떤 커넥터를 쓰면 될지 제안해줘."

예) "클로드 커넥터로 매일 AI 이슈를 개인 공간에 정리하는 자동화를 할건데 이중에서 어떤 커넥터를 쓰면 될지 제안해줘."

→ Claude가 화면 속 커넥터 목록을 분석해서 목적에 맞는 최적의 커넥터를 추천해줄 거예요!

✍️ STEP 3. 프롬프트 작성 — 커넥터 추가 후 프롬프트 요청하기

요약 : 추천받은 커넥터를 추가한 뒤, 작업 프롬프트를 만들어달라고 요청하세요.

Step 1. 커넥터 추가하기

  1. 클로드 앱 좌측 상단의 Customize 버튼 클릭 → 커넥터 섹션으로 이동 → 커넥터 둘러보기 클릭
  2. 필요한 커넥터를 찾아 추가 클릭→ 로그인 또는 연결 완료 후 커넥터 목록에 추가되었는지 확인
    예시

    웹 또는 데스크톱 목록에 추가가 되어있는지 확인하세요.

    만약 ‘연결되지 않음’ 목록에 뜬다면 제대로 연결되지 않았다는 의미입니다.

Step 2. 프롬프트 작성 요청하기

커넥터 추가가 완료되면, 클로드에게 예정된 작업에 맞는 프롬프트를 만들어달라고 요청하세요.

💬 프롬프트 예시
💡
"이런 커넥터를 추가했어. 예정된 작업의 프롬프트를 작성해줘. 커넥터도 명시해."

예) "GitHub와 Notion 커넥터를 추가했어. 매일 오전 9시에 GitHub의 새 이슈를 가져와서 Notion에 정리하는 작업의 프롬프트를 작성해줘. 커넥터도 명시해."

클로드가 해당 커넥터들을 활용하는 완성된 자동화 프롬프트를 만들어줄 거예요. 이 프롬프트를 복사해두세요!

부체오 추가 TIP
  • 프롬프트에 클로드가 할 행동, 출력형식, 규칙 등이 적혀있는지 확인하세요.
  • 프롬프트 내의 목적지 명칭(예 : 노션 페이지명, 슬랙 채널명)이 실제 설정된 명칭과 일치하는지 확인하세요. (예: 페이지명이 'AI뉴스'인 경우, 프롬프트에 ‘AI뉴스’ 페이지가 목적지임을 명시)
💡
[예시]

# 행동 :

"AI 뉴스 아카이브" 페이지의 표에 'https://www.aitimes.com/' 오늘 뉴스를 행으로 추가해 주세요.

# 출력형식 (notion 페이지) :

  • 날짜 (오늘 날짜 자동 입력)
  • 제목
  • 핵심 요약 (1~2줄)
  • 카테고리 (예: LLM, 로보틱스, 규제, 스타트업 등)
  • 출처 링크

# 규칙 :

기존 행은 건드리지 말고, 새 행만 추가해 주세요.

⏰ STEP 4. 실행 설정 — '예정됨'에서 자동화 완성!

요약 : 프롬프트를 복사해서 반복 실행을 예약하세요.

자동화 등록 순서

  1. 프롬프트 복사 후 클로드 데스크톱 앱 → Cowork→ 좌측 메뉴에서 예정됨(Scheduled) 클릭
  2. New Task 버튼 클릭 → 설명 칸 아래에 복사한 프롬프트 붙여넣기
  3. 빈도 설정 (시간별 / 매일 / 평일 / 주간 등 원하는 주기 선택) → 저장 버튼 클릭
  4. 자동 실행 시작!

이제 설정한 주기마다 클로드가 자동으로 작업을 수행하고 결과를 지정된 곳에 저장합니다!

💡 TIP

1. 권한 승인 절차 생략 방법

  • 자동화 실행할 때마다 권한 허용을 눌러야 하는 경우가 있는데, 작업 설정 하단의 '질문' 모드를 '묻지 않고 실행'으로 변경하면 됩니다. 매번 승인 버튼을 누를 필요 없이 즉시 작업이 수행됩니다.
  • ⚠️ 주의: Claude가 확인 없이 데이터를 처리하므로, 보안이 중요한 작업 시에는 신중하게 설정하세요.

2. 자동화 실행 필수 조건

  • 자동화가 실행되려면 컴퓨터가 켜져있는 상태여야 합니다.
  • 절전 모드나 앱 종료 시 작업은 스킵되며, PC 복귀 또는 앱 재실행 시 자동으로 다시 시작됩니다.
