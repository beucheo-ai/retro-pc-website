1. 이게 뭔가요? — 개념 요약

  • LLM Wiki: AI가 대신 써주고 유지·관리하는 개인 지식 위키. 자료를 넣을수록 지식이 쌓이고 연결되는 복리 구조.
  • 3개 층 구조
    • raw/ — 원본 자료 (⚠️ 읽기 전용. AI가 절대 수정·이동·삭제 금지)
    • wiki/ — AI가 쓰고 관리하는 마크다운 페이지들
    • CLAUDE.md — 위키 규칙·워크플로를 정의한 스키마 (계속 다듬는 대상)
  • 보조 파일: index.md(전체 페이지 목록), log.md(작업 기록)
  • 3가지 작업 루프: Ingest(자료 투입) → Query(질문·종합) → Lint(월 1회 건강검진)
  • 최종 결과물: 이 위키를 초보자도 클릭하며 탐색할 수 있는 나무위키 스타일 HTML 페이지로 변환 (4단계)

  • ☐ 백업: vault 폴더 통째로 복사 또는 git init ← AI에게 맡기기 전 필수

3. 위키 세팅

이 단계에서 하는 일을 한마디로 하면 "파일을 정리하는 것"입니다 — 여기저기 흩어진 자료(또는 앞으로 모을 자료)를 raw/wiki 구조로 넣고, AI가 알아서 정리·요약·연결하게 만드는 과정이에요.

아래 두 경우로 나뉘어 있으니, 본인 상황에 맞는 쪽을 골라 진행하면 됩니다. 둘 다 진행 순서는 비슷하지만 시작점이 다릅니다.

  • 3-1. 기존 옵시디언 사용자 — 노션, 메모장, 문서 파일 등 이미 정리 안 된 원본 자료가 어느 정도 쌓여 있는 경우. 자료를 raw/ 폴더에 옮기는 것부터 시작합니다.
  • 3-2. 옵시디언이 처음인 경우 — 아직 모아둔 자료가 없고, 옵시디언도 처음 써보는 경우. 빈 구조만 먼저 만들고, 자료는 앞으로 하나씩 쌓아갑니다.

👉 둘 다 정답이 없으니, 지금 내 상황에 더 가까운 쪽 하나만 골라 그 파트를 따라가면 됩니다. (헷갈리면: raw/에 넣을 파일이 이미 10개 이상 있다 → 3-1 / 없다 → 3-2)

3-1. 기존 옵시디언 사용자 (raw 자료가 이미 있는 경우)

STEP 1. 원본 raw 폴더에 옮기기

  • vault 안 raw/ 폴더 생성 후 기존 파일 넣기
  • ⚠️ 비밀번호·API 키·토큰 같은 민감정보는 옮기기 전에 빼기 — raw에 들어가는 순간 Claude가 읽을 때마다 서버로 전송되므로, 애초에 넣지 않는 게 가장 안전

STEP 2. 위키 세팅 규칙 설계 🔴 Fable 5

추천 모드: Manual (또는 Plan mode) · 추천 Effort: high

가장 중요한 순간. 여기서 잡힌 구조 위에 모든 게 쌓임.

나는 [내 정보 입력]야. Karpathy의 "LLM Wiki" 패턴으로
내 옵시디언 개인 지식 위키를 세팅하려 해. 이번 세션은 "구조 설계·합의 단계"까지만 진행한다.

패턴 핵심(참고: https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f):
- 3계층: raw(원본, 불변) → wiki(LLM이 생성·유지하는 md 페이지) → schema(CLAUDE.md, 운영 규칙)
- 운영: ingest / query / lint
- index.md는 내용 카탈로그, log.md는 시간순 append 기록

이번에 해줄 것:
1. raw/ 파일들을 읽고 어떤 주제가 있는지 파악해서, 주제 목록부터 먼저 알려줘(그다음 근거).
2. 내 자료에 맞는 위키 폴더 구조와 CLAUDE.md(스키마) 초안을 제안해줘. 각 구분은 왜 그렇게 나눴는지 한 줄씩만.
3. index.md·log.md 초안도 같이 제안해줘.

경계:
- raw/는 읽기 전용. 수정·이동·삭제·이름변경 금지.
- 실제 위키 페이지 생성·내용 정리는 아직 하지 마.
- 파일을 디스크에 쓰기 전에 나와 구조를 먼저 합의한다. 이번엔 제안만 내놓고 내 확정을 기다려.
- 요청 범위를 넘는 것(검색 CLI, 추가 페이지 타입, 자동화 스크립트 등)은 지금 만들지 마. 최소 구조로.
- raw/에서 비밀번호·API 키 등 민감정보를 발견하면 위키·log에 옮기지 말고 나에게만 보고해.

출력: 주제 목록 → 제안 폴더 트리 → CLAUDE.md/index.md/log.md 핵심 초안. 결과부터, 읽기 쉽게.

💡 제안이 나오면 옵시디언에서 CLAUDE.md 초안을 열어 wiki/ 섹션의 분류를 확인하세요. 이 분류가 나중에 나무위키 사이트의 카테고리로 그대로 이어지기 때문에, 마음에 안 드는 부분은 이 단계에서 피드백을 주고 마음에 들 때까지 다듬은 뒤 확정하는 게 좋습니다.

STEP 3. 첫 ingest 10~20개 🔴 Fable 5

  • 세팅 규칙을 바탕으로 파일 정리 시작
  • 첫 정리는 본보기용으로 'fable5'에게 시키기
합의한 위키 폴더 구조·CLAUDE.md 스키마에 따라 raw/ 파일을 ingest한다.
목적: raw 폴더 원본을 검색·재사용 가능한 위키로 쌓아 내 지식베이스를 만든다.

파일 하나당:
- 읽고 핵심 요약 페이지를 위키에 생성
- 관련 주제 페이지를 생성/갱신하고 요약↔주제 양방향 링크 연결
- index.md(카탈로그)와 log.md(시간순)에 반영

진행 방식:
- 이번엔 raw/ 파일 5개만 처리하고 멈춘 뒤 결과를 보여줘. 내가 확인하면 이어서 진행.

보고: 처리한 파일 5개 각각 → 만든/갱신한 페이지와 링크 → index·log 변경점. 결과부터 간결히.

💡 5~10개씩 끊어서 진행. 피드백 내용이 있다면 입력 후 CLAUDE.md 수정.

STEP 4. 남은 대량 ingest 🔵 Sonnet

추천 모드: Auto mode · 추천 Effort: high

피드백이 없다면 이제 나머지 정리는 하위 모델로 충분. Sonnet으로 전환 후 STEP 3과 같은 요청 반복.

raw 폴더의 파일들을 하나씩 ingest 해줘. 파일마다:
- 읽고 핵심 요약 페이지를 위키에 만들고
- 관련 주제 페이지를 만들거나 업데이트하고 (서로 링크 걸기)
- index.md와 log.md 갱신
이제 멈추지 말고 계속 진행하자.
💡
여기까지가 파일 정리 끝!

아래 5~6번은 이 옵시디언 위키를 꾸준히 운영하는 요령이에요. 바로 웹사이트부터 만들고 싶다면 [4. 나무위키 페이지 만들기 🌳]로 건너뛰세요.

3-2. 옵시디언이 처음인 경우

이 파트는 "옵시디언이 뭔지 잘 모르겠고, 아직 모아둔 자료도 없는" 분들을 위한 것입니다. 미리 알아두면 좋은 것부터 짚고 갈게요.

  • Obsidian(옵시디언): 마크다운(.md) 파일로 메모를 쌓아가는 무료 앱. 어렵게 생각할 것 없이, 그냥 '메모장 + 폴더'라고 보면 됩니다. obsidian.md에서 설치.
  • vault(볼트): 옵시디언에서 메모들을 모아두는 폴더 하나. 앱 설치 후 "새 vault 만들기"로 폴더 하나를 지정하면 그게 끝입니다.
  • 자료가 하나도 없어도 전혀 문제 없습니다. 이 3-2 루트는 애초에 '자료 없는 상태'에서 출발하도록 설계되어 있어요. 처음엔 껍데기(폴더 구조)만 만들고, 자료는 살면서 자연스럽게 채워나가면 됩니다.

STEP 1. 빈 구조만 먼저 🔴 Fable 5

Claude Code(또는 Fable 5 연결된 채팅)에 아래 프롬프트를 그대로 붙여넣고, [ ] 부분만 내 얘기로 채우면 됩니다. 이 한 번의 대화로 vault 안에 raw/, wiki/ 폴더와 CLAUDE.md, index.md, log.md 파일들이 자동으로 만들어져요 — 폴더를 직접 만들 필요는 없습니다.

카파시의 "LLM Wiki" 패턴으로 지식 위키를 시작하려고 해. https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f
나는 (자기 소개: 직업/관심 주제/목적).
아직 자료는 없어. 앞으로 이런 걸 모을 예정이야: (예: AI 트렌드 기사, 유튜브 노트, 책 요약)
raw/, wiki/ 폴더와 CLAUDE.md, index.md, log.md 초기 버전을 만들어줘.
주제 분류는 최소한으로만 잡고, 자료가 쌓이면 같이 발전시키자.

💡 지금 단계에서는 완벽한 분류 체계를 고민할 필요가 없습니다. 데이터가 없는데 구조부터 정교하게 짜면 나중에 다시 뜯어고치게 될 확률이 높아요. 자료 5~10개가 쌓인 뒤 구조를 다시 잡는 게 정석이니, 지금은 최소한의 뼈대만 만든다고 생각하세요.

STEP 2. 자료 수집 습관 만들기 (사람이 직접 하는 단계)

AI가 정리를 도와주려면, 일단 정리할 '원본'이 있어야 합니다. 아래처럼 평소에 보는 것들을 그때그때 raw/ 폴더에 파일로 저장하는 습관을 들이면 됩니다.

  • Obsidian Web Clipper: 브라우저에 설치하는 무료 확장 프로그램. 웹에서 읽던 글이 있으면 버튼 한 번 눌러서 그 페이지 내용을 마크다운 파일로 저장해줍니다 → 저장 위치를 vault의 raw/ 폴더로 지정해두면 자동으로 쌓입니다.
  • 웹 글이 아니어도 괜찮습니다 — 유튜브 보고 든 생각, 강의 노트, 회의 메모, 문득 떠오른 아이디어도 메모장에 옮겨 적어 raw/에 파일 하나로 저장하면 됩니다. (형식은 자유롭습니다. 완벽하게 쓰지 않아도 돼요.)
  • ⚠️ 비밀번호·API 키 같은 민감정보는 클리핑·메모에서 미리 제외하세요.

STEP 3. 하나씩 ingest 🔴 Fable 5 (초반 10~20개까지)

여기서 'ingest'란 어려운 말이 아니라, "방금 raw/에 넣은 파일 하나를 AI가 읽고, 요약 페이지로 만들고, 관련 주제와 연결해서 wiki/ 안에 정리해두는 것"을 뜻합니다. 파일 하나 넣을 때마다 아래처럼 짧게 요청하면 됩니다.

raw에 새 파일 넣었어. ingest 해줘.

초반 10~20개 정도는 매번 결과를 직접 읽어보고, 마음에 안 드는 부분이 있으면 그 피드백을 CLAUDE.md에 반영해주세요 (예: "요약은 3줄을 넘지 않게 해줘" 같은 식으로 말하면 AI가 알아서 규칙으로 정리해줍니다). 이렇게 20개쯤 지나서 스타일이 안정되면, 그 뒤로는 더 저렴한 🔵 Sonnet으로 전환해서 같은 요청을 반복해도 충분합니다.

STEP 4. 구조 재정비 🔴 Fable 5 (자료 20~30개 시점, 1회)

자료가 어느 정도 쌓이면, 처음에 최소한으로 잡았던 분류가 지금 자료량에 잘 맞는지 한 번 점검할 시점입니다.

지금까지 쌓인 위키를 보고 주제 분류가 적절한지 검토해줘.

4-2. 업데이트 스킬 만들기

다 되었으면 앞으로 위키 업데이트 시 사이트에도 반영되도록 해야겠죠? 매번 길게 설명할 필요 없이 명령어 한 번으로 업데이트되도록 스킬을 제작해달라고 합니다.

위 과정을 매번 설명할 필요 없이 슬래시 명령 한 번으로 실행되도록 "wiki 폴더에서 수정 시각이 HTML보다 최신인 노트를 찾아, 그 문서들만 나무위키 스타일로 다시 써서 반영해라"라는 절차를 포함한 스킬을 제작해줘.

앞으로 위키 업데이트 시 채팅에 만든 명령어(예: /wiki-sync)만 입력하면 끝!

5. 안전 수칙 (반드시)

  1. raw는 불변 — AI는 읽기만. 이 규칙을 CLAUDE.md 맨 위에 명시
  2. 시작 전 백업 — git 또는 폴더 복사. 이후는 git이 자동 안전망
  3. 추가는 자유, 변경은 승인 — 새 파일 생성은 위험 없음. 기존 노트 이동·수정·삭제는 반드시 확인 후
  4. 초반엔 기본 권한 모드 — 승인하며 관찰하다가, 신뢰 쌓이면 자동 승인으로
  5. 비밀번호·API 키 등 민감정보는 위키로 옮기지 않기 — raw에서 발견되면 위키·log에 기록하지 말고 사용자에게만 보고하도록 CLAUDE.md에 규칙으로 명시
